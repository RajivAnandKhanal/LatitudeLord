## landing page for user without authentication
-> shows the buses near the user
-> if user clicks on specific bus then mini interface opens and shows the estimated time of arrival of that bus (only via the gps calculation)
-> search by bus station -> in this feature user selects bus and station (filteration happens by -> user entered bus station to last station of the following bus) 

## landing page for user with authentication (only as normal not the bus staff user) 
-> shows the buses near the user
-> can chat with bus staff
-> if user clicks on specific bus then mini interface opens and shows estimated time of arrival of that bus (via gps and also integrates the machine learning data if data present in backend)
-> collects the data (of the time taken and gps location of the journey) of user if user gets into the specific bus
-> search by bus station -> in this feature user selects bus and station (filteration happens by -> user entered bus station to last station of the following bus) 



## Registration
-> first asks for whether the user is bus staff or normal user

-> asks for 
Driver Name (Compulsory)
Driver Phone Number
Bus Number Plate (Compulsory)
Bus Number (In case if a company have multiple bus) 
Bus Staff Name
Bus Staff Phone Number (Compulsory)
Bus Route Of 7 Days of weak (Compulsory)
Driver Photo (Compulsory)

## Registration if user is normal user

User's Name (Compulsory)
User's Email (Compulsory)
User's Gender

USer's Photo (Compulsory)
